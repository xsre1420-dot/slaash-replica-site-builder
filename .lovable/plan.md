
# خطة بناء منصة SaaS لشركات التوصيل — 7 مراحل

## المرحلة 1: قاعدة البيانات والأدوار (Database & RBAC)
- إنشاء جدول `tenants` (شركات التوصيل) مع بيانات العلامة التجارية
- إنشاء جدول `user_roles` مع enum للأدوار (super_admin, company_admin, merchant, driver)
- إنشاء جدول `drivers` مع حالة السائق وموقعه
- إنشاء جدول `subscriptions` للاشتراكات
- إضافة `tenant_id` للجداول المتأثرة (stores, orders, products)
- ربط المتاجر الحالية بشركات التوصيل عبر `tenant_id`
- إنشاء دوال `SECURITY DEFINER` للتحقق من الأدوار
- سياسات RLS محدثة لعزل بيانات كل شركة

## المرحلة 2: لوحة تحكم شركة التوصيل (Company Dashboard)
- صفحة تسجيل دخول/تسجيل خاصة بشركات التوصيل
- لوحة تحكم رئيسية مع إحصائيات (طلبات اليوم، سائقين نشطين، إيرادات)
- إدارة التجار: إضافة/تعديل/حذف تجار مرتبطين بالشركة
- عرض جميع طلبات تجار الشركة
- إعدادات الشركة (الشعار، الألوان، النطاق الفرعي)

## المرحلة 3: نظام السائقين (Driver System)
- إضافة وإدارة السائقين من لوحة الشركة
- تعيين الطلبات للسائقين
- واجهة السائق (موبايل): عرض الطلبات المسندة، تحديث الحالة
- تتبع حالات التوصيل (تم الاستلام، في الطريق، تم التسليم)

## المرحلة 4: تدفق الطلبات المتكامل (Order Flow)
- ربط الطلب تلقائياً بشركة التوصيل عند إنشائه
- إشعارات لحظية للتاجر والسائق والعميل
- تتبع الطلب للعميل (صفحة تتبع عامة)
- حساب رسوم التوصيل حسب شركة التوصيل

## المرحلة 5: لوحة Super Admin
- إدارة جميع شركات التوصيل
- إدارة الاشتراكات والفواتير
- إحصائيات شاملة للمنصة
- إعدادات النظام العامة

## المرحلة 6: نظام الاشتراكات والدفع
- باقات (مجاني، أساسي، متقدم) مع حدود
- تتبع الاستخدام (عدد الطلبات، التجار، السائقين)
- تكامل بوابة الدفع

## المرحلة 7: ميزات متقدمة
- إشعارات SMS/WhatsApp
- تحليلات متقدمة
- API خارجي للتكامل
- دعم تعدد شركات التوصيل للتاجر الواحد
